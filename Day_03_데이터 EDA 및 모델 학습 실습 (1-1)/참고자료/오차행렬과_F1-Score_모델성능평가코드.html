오차 행렬과 F1-Score

학습목표

  • 오차행렬에 대해 이해할 수 있다.
  • 모델의 성능을 파악하는 지표에 대해 이해할 수 있다.

오차행렬 (Confusion Matrix)

모델이 생성된 뒤에는 모델의 성능을 측정해야 합니다.

분류 모델의 결과를 효과적으로 표현하는 방법으로 오차행렬(Confusion Matrix)을 사용할 수 있습니다.

분류 모델에서 모델은 1과 0의 값을 분류하고 이를 실제 값과 비교합니다.

오차행렬(Confusion Matrix)은 모델이 예측한 값과 실제 값을 쉽게 비교하기 위한 것으로 총 4개의 영역으로 이루어져 있습니다.

그림처럼 모델이 예측한 값과 실제 값을 행렬로 표현하여 성능 평가에 활용할 수 있습니다.

  • 모델이 1로 예측한 값이 정말 1인 경우 TP(True Positive)라고 합니다.
  • 모델이 1로 예측했지만 사실 0인 경우 FP(False Positive)라고 합니다.
  • 모델이 0으로 예측했지만 사실 1인 경우 FN(False Negative)라고 합니다.
  • 모델이 0으로 예측한 값이 정말 0인 경우 TN(True Negative)라고 합니다.

정확도, 정밀도, 재현율, F1-Score

오차행렬에서 표현된 값들을 기반으로 정확도(Accuracy), 정밀도(Precision), 재현율(Recall), F1-Score를 구할 수 있습니다.

정확도(Accuracy)

정확도는 전체 데이터 수 중에서 올바르게 예측된 데이터의 수의 비율입니다. TP와 TN을 더한 값을 4개의 영역의 합산으로 나누어 구할 수 있습니다.

정확도는 데이터에 따라 편향 될 수 있습니다. 예를 들어 희귀병을 예측하는 모델에서 모두 병이 없다고 예측해버리면 높은 정확도를 가질 수 있습니다. 모델이 모두 병이 없다고 한다면 전부 Negative로 판단할 것이고, FN과 TN에만 숫자가 할당됩니다.

TP와 FN의 비중이 작기 때문에 TP가 0이라도 TN에 많은 숫자가 할당되어 높은 정확도가 나타납니다.

Point

정확도가 높더라도 항상 병이 없다고 예측하는 모델은 의미가 없습니다.

재현율(Recall)

위의 상황에서는 재현율(recall)이 도움을 줄 수 있습니다. 재현율(recall)은 실제 값이 Positive(1)인 것 중 모델이 Positive(1)라고 예측한 비율을 나타냅니다. 만약 모델이 전부 병이 없다고 예측한다면 TP에 값이 없기 때문에 재현율 값이 0이 됩니다.

하지만 재현율 또한 완벽한 통계치가 아닙니다. 만약 모델이 전부 병이 있을 것이라고 예측한다면 재현율은 1이 됩니다. 재현율만 높이기 위한 모델을 만들면 모델은 전부 병이 있을 것이라고 예측할 것입니다.

Point

항상 병이 없다고 예측하는 모델과 마찬가지로 항상 병이 있다고 예측하는 모델도 의미가 없습니다.

정밀도(Precision)

정밀도(precision)은 모델이 Positive(1)로 예측한 값 중 실제 값이 Positive(1)인 비율을 나타냅니다.

만약 모델이 전부 병이 있다고 예측한다면 정밀도 값이 매우 낮아지게 될 것입니다.

F1-Score

정확도와 재현율, 정밀도를 모두 사용한다면 모델의 성능을 잘 표현할 수 있습니다. 하지만 하나의 대표값으로 사용하기 위해 만든 지표가 바로 F1-Score 입니다.

F1-Score는 정밀도와 재현율의 조화평균입니다.

정밀도와 재현율을 조합하여 만든 지표이기 때문에 두 가지 수치가 모두 높아야 F1-Score도 높은 수치를 가질 수 있습니다.

Point

F1-Score는 분류 모델의 최종 성능 평가 지표로 자주 활용됩니다. F1-Score와 더불어 정확도, 정밀도, 재현율과 같은 지표들도 잘 이해한다면 모델에 대한 적합한 평가를 할 수 있습니다.

성능 평가 지표와 오차행렬

성능평가 지표와 오차행렬을 생성하기 위해 sklearn.metrics 모듈에서 confusion_matrix, accuracy_score, precision_score, recall_score, f1_score 기능들을 불러올 수 있습니다.

Guide

패키지를 불러온 뒤 테스트용 Input 특성을 활용하여 예측값을 생성합니다. grid_search모델에 우리가 찾은 최적의 하이퍼파라미터 값이 적용된 모델이 저장되어 있습니다. grid_search 모델에 predict함수를 사용하여 준비된 테스트용 input특성을 적용하고 새로운 변수에 할당합니다.

아래 함수에 실제 테스트 값과 예측한 테스트 값을 넣어 해당 값들을 생성할 수 있습니다.

  • 정확도 : accuracy_score
  • 정밀도 : precision_score
  • 재현율 : recall_score
  • F1-score : f1_score
  • 오차행렬 : confusion_matrix
from sklearn.metrics import confusion_matrix, accuracy_score, precision_score, recall_score, f1_score

predictions = grid_search.predict(테스트용 Input특성들)

print("정확도: ", accuracy_score(y_test, predictions))
print("정밀도: ", precision_score(y_test, predictions))
print("재현율: ", recall_score(y_test, predictions))
print("F1-Score: ", f1_score(y_test, predictions))
print("오차행렬: \n", confusion_matrix(y_test, predictions))
