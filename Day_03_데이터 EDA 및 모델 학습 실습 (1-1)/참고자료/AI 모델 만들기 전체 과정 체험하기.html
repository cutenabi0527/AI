📊 AI 모델 만들기 전체 과정 체험하기

AI 모델을 만드는 것은 '데이터를 보고 규칙을 학습하는 기계'를 만드는 여정과 같습니다.
아래 단계를 따라가며 데이터가 어떻게 모델로 변신하는지 직접 확인해보세요!

1단계: 데이터 준비 (학습용과 시험용 분리)

🤔 왜 데이터를 나눠야 할까요?
모델을 공부시킬 '연습문제(Train Set)'와, 공부가 끝난 모델을 평가할 '실제 시험(Test Set)'으로 나누는 과정입니다. 만약 연습문제로 시험을 보면, 답을 외워서 100점을 맞을 순 있지만 새로운 문제를 풀 순 없겠죠? 모델이 얼마나 '일반화'되었는지 공정하게 평가하기 위해 데이터를 반드시 분리해야 합니다.

2단계: 데이터 다듬기 (단위 맞추기)

🤔 왜 단위를 맞춰야 할까요?
만약 '키(170cm)'와 '나이(25세)'라는 두 가지 특성이 있다면, 모델은 숫자 값이 훨씬 큰 '키'에 더 많은 영향을 받게 됩니다. 이런 불공평을 막기 위해 모든 특성들이 비슷한 범위의 값을 갖도록 '단위'를 맞춰주는 과정입니다. 스케일링을 통해 모델은 각 특성을 공평하게 바라보고 더 안정적으로 학습할 수 있습니다.

3단계: 모델 학습 (경사 하강법으로 규칙 찾기)

🤔 '학습'이란 무엇일까요? (경사 하강법 시뮬레이션)
이 단계의 목표는 데이터의 두 클래스를 가장 잘 나눌 수 있는 **최적의 '구분선(Decision Boundary)'**을 찾는 것입니다. 이 과정을 실제로 확인하기 위해, 머신러닝의 핵심 학습 알고리즘인 **경사 하강법(Gradient Descent)**을 시뮬레이션합니다.
  1. 아무렇게나 그은 무작위 선에서 학습을 시작합니다.
  2. **'학습 실행'** 버튼을 누르면, 모델은 현재 선이 얼마나 데이터를 잘못 예측했는지(오류, Loss) 계산합니다.
  3. 오류를 가장 빨리 줄일 수 있는 방향으로 선을 아주 약간씩(Learning Rate) 움직입니다.
  4. 이 과정을 반복하면, 선은 점차 두 데이터를 가장 잘 나누는 최적의 위치로 이동하게 됩니다. 오른쪽 그래프는 학습이 진행될수록 오류(Loss)가 줄어드는 것을 보여줍니다.

4단계: 모델 평가 (성적표 확인)

🤔 모델의 성능, 어떻게 숫자로 표현할까요?
학습된 모델(구분선)을 평가하기 위해, 먼저 각 데이터가 특정 클래스에 속할 **'확률'**을 계산해야 합니다. 이때 **시그모이드(Sigmoid) 함수**가 사용됩니다. 어떤 점과 구분선 사이의 거리를 이 함수에 넣으면, 0~1 사이의 확률값이 나옵니다. 이 확률을 아래 슬라이더의 **Threshold(임계값)**와 비교하여 최종적으로 0 또는 1로 예측합니다. 예측 결과 차트의 주황색 점선은 현재 Threshold를 나타내는 **'확률 등고선'**입니다.

ROC 커브 vs. PR 커브
두 그래프 모두 임계값을 바꿔가며 모델의 성능을 종합적으로 보여주지만, 관점이 다릅니다. 페이지 상단의 '어려운 데이터' 옵션을 켜면 두 그래프가 왜 교과서처럼 곡선으로 그려지는지 확인할 수 있습니다.
  • ROC 커브: 전체 클래스 비율이 균등할 때 유용합니다. Y축(TPR)은 '실제 정답을 얼마나 잘 맞췄는가'(이득), X축(FPR)은 '실제 오답을 얼마나 정답으로 착각했는가'(비용)를 나타냅니다.
  • PR(Precision-Recall) 커브: 클래스 불균형이 심할 때(예: 암 진단, 불량품 검사) 더 유용합니다. Y축(Precision)은 '모델이 정답이라고 예측한 것 중 실제 정답 비율'(정확성), X축(Recall)은 '실제 정답 중 모델이 정답이라고 예측한 비율'(재현율)을 나타냅니다.

번외: 데이터 탐색을 위한 다른 기법들

모델링 외에도 데이터의 숨겨진 특징을 파악하기 위한 여러 기법들이 있습니다.

5. 차원 축소 (핵심 특징만 뽑아보기)

🤔 차원 축소는 왜 할까요?
특성이 수십, 수백 개가 되면 사람이 데이터를 이해하기 어렵습니다. PCA(주성분 분석)는 여러 특성에 흩어진 정보의 손실을 최소화하면서, 데이터를 가장 잘 설명하는 새로운 '핵심 축' 몇 개를 뽑아내는 기술입니다. 3차원 물체를 2차원 '그림자'로 표현해도 핵심 형태를 알아볼 수 있는 것과 같습니다. 데이터 시각화나 모델 학습 속도 개선에 유용합니다.

6. 군집화 (숨겨진 그룹 찾아내기)

🤔 군집화는 언제 사용할까요?
정답(레이블)이 없는 데이터에서 비슷한 특성을 가진 데이터끼리 '숨겨진 그룹'을 찾아내는 '비지도 학습'의 한 종류입니다. K-Means 알고리즘의 작동 방식을 아래에서 단계별로 확인해봅시다.
1. 할당(Assign): 모든 점을 가장 가까운 중심점에 할당합니다.
2. 업데이트(Update): 각 그룹의 중심점을 그룹에 속한 점들의 평균 위치로 이동합니다.
이 두 과정을 중심점이 더 이상 변하지 않을 때까지 반복합니다.
